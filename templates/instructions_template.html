/

Root endpoint '/' muestra opciones de la API
GET: te trae a esta pagina.
POST: devuelve un JSON con todas las opciones de la API

- Request [POST]

no necesita body

- Response

Este endpoint devuelve un JSON con cada endpoint y sus propiedades

Si hay fallos en la NO HAY FALLBACK

/login

Login endpoint, se encarga de autenticar al usuario.

- Request [POST]

Recibe dos parametros en el body, user y passw

- Response

Setea 2 cookies, admin y token
"token" se encarga de la autenticacion del usuario en el backend
"admin" es una cookie para la identificacion de los adminsitradores en el frontend
estructura del token:

El token se compone de dos partes, separadas por un punto(.).
A la izquierda, el nombre de usuario, a la derecha una string de 50 caracteres auto-generados aleatoreamente

Si hay fallos en la autenticacion devuelve un JSON

{"auth error"}

/user

User endpoint, se encarga manejo de cuentas de empleados.

- Request [GET]

Recibe 0 o 1 parametro en la URL

- Response

Este endpoint devuelve un JSON
Si la cuenta fue creada de manera correcta devuelve id y name

Si no encuentra el usuario devuelve un objeto vacio: {}

- Request [POST]

Recibe 1 parametro en el body

- Response

Este endpoint devuelve un JSON
Si la cuenta fue creada de manera correcta devuelve id, name y passw
la contraseña es auto-generada en el backend

- Request [PATCH]

recibe dos o mas parametros
user (nombre de usuario){no modificable, se envia como referencia}
passw (nueva contraseña usuario)
active(desactiva la cuenta {0})
admin (da permisos de administrador {1})

- Response

devuelve un mensaje de error o exito.

- Request [DELETE]

recibe 1 parametro

- Response

devuelve un objeto con los datos del usuario desactivado
