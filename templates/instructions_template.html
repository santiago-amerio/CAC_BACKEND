/

Root endpoint '/' muestra opciones de la API
GET: te trae a esta pagina.
POST: devuelve un JSON con todas las opciones de la API

- Request [POST]

no necesita body

- Response

Este endpoint devuelve un JSON con cada endpoint y sus propiedades

Si hay fallos en la NO HAY FALLBACK

/singup

Singup endpoint, se encarga de registrar una nueva cuenta.

- Request [POST]

Recibe dos parametros en el body, user y passw

- Response

Este endpoint devuelve un JSON
Si la cuenta fue creada de manera correcta devuelve id y name

Si hay fallos en la NO HAY FALLBACK

/login

Login endpoint, se encarga de autenticar al usuario.

- Request [POST]

Recibe dos parametros en el body, user y passw 

                        {
                                "user": "nombre_de_usuario",
                                "passw": "contraseña_en_texto_plano"
                                }

- Response

SHOULD SET A COOKIE

Establece una cookie en el cliente, con la key "token" y el valor del token

El endpoint login responde con JSON.
Si la cuenta fue autenticada correctamente, devuelve un token. 

                        {token:"test123.eeaykbvixaytjksgyddbzgyxehqhvrydjcvhigwiikrskbanoj"}

El token se compone de dos partes, separadas por un punto(.).
A la izquierda, el nombre de usuario, a la derecha una string de 50 caracteres auto-generados aleatoreamente

Si hay fallos en la autenticacion devuelve un JSON

{"auth error"}
